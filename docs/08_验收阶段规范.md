# 验收阶段规范

验收阶段只看证据，不看意图。

## 必验项

- 构建通过。
- 服务启动通过。
- 前端页面可访问。
- 后端健康检查可访问。
- 错误配置能拒绝启动或明确报错。
- 核心接口符合契约。
- 正式链路无 Mock。
- 调试入口关闭或可删除。
- deployment_control 评估完成。
- 部署和回滚说明存在。

## 验收输出

验收报告必须分三类：

- 已通过。
- 部分完成。
- 未完成或风险项。

不得把“计划完成”写成“已验证完成”。

## 无 Mock 检查

验收时必须检查：

- 生产前端路由。
- 生产后端路由。
- 生产构建产物。
- 配置文件。
- deployment_control 发布包。

出现 mock、simulator、fake、fallback、debug 等关键字时，必须说明是否属于隔离调试能力。

## 门禁

只有所有高风险项都有结论时，才能写“可进入交付”。现场未验证必须写明“未现场验证”。
