# 开发阶段规范

开发阶段按接口契约实现正式链路。目标是小步提交、逐项验证。

## 开发顺序

1. 固化接口契约。
2. 建立配置校验。
3. 实现后端真实接口。
4. 实现前端最小页面。
5. 接入状态和错误反馈。
6. 补充健康检查。
7. 更新文档和验收清单。

## 正式链路要求

- 接口返回真实状态。
- 控制动作必须有权限、互锁、确认和反馈。
- 后端必须记录关键日志。
- 前端不得假定动作成功。
- 配置错误必须拒绝启动或明确报错。

## 接口契约

每个接口至少定义：

- URL
- 方法
- 请求字段
- 响应字段
- 错误码
- 超时行为
- 是否可重试
- 是否涉及真实设备控制

## 禁止项

- 禁止假成功。
- 禁止捕获错误后静默返回正常。
- 禁止在正式服务中混入临时 Mock。
- 禁止多个配置文件表达同一个现场参数。
- 禁止先写 UI 再倒推接口。

## 门禁

进入调试阶段前必须确认：

- 构建通过。
- 核心接口能被调用。
- 配置校验存在。
- 错误路径有反馈。
- 文档与接口一致。
