# 项目目录结构规范

本规范定义 AI 生成新工业 HMI / 工控项目时的推荐目录。目录是约束，不是装饰。

## 1. SimpleHmi_WEILI 仓库目录

SimpleHmi_WEILI 自身推荐结构：

```text
SimpleHmi_WEILI/
├── AGENTS.md
├── README.md
├── SIMPLE_HMI.md
├── ENGINES.md
├── LICENSE
├── docs/
├── templates/
└── deployment_control/
```

本仓库只放规范、模板和最小部署控制入口，不放客户业务代码。

## 2. AI 生成新项目目录

新工业项目推荐结构：

```text
project-name/
├── AGENTS.md
├── README.md
├── docs/
│   ├── 00_INDEX.md
│   ├── requirements.md
│   ├── architecture.md
│   ├── api-contract.md
│   ├── deployment.md
│   └── acceptance.md
├── apps/
│   ├── web-hmi/
│   └── api/
├── packages/
│   ├── contracts/
│   └── adapters/
├── deployment_control/
├── simulator/
└── scripts/
```

## 3. 目录职责

| 目录 | 职责 | 禁止 |
| --- | --- | --- |
| `docs/` | 需求、架构、接口、部署、验收 | 放运行代码 |
| `apps/web-hmi/` | 前端 HMI | 混入调试 Mock 页面 |
| `apps/api/` | 后端 API | 假成功、静默降级 |
| `packages/contracts/` | 前后端共享契约 | 写业务实现 |
| `packages/adapters/` | 设备和外部系统适配接口 | 强绑定具体客户逻辑 |
| `deployment_control/` | 部署控制能力 | 写业务决策 |
| `simulator/` | 用户明确允许时的临时模拟能力 | 默认进入生产 |
| `scripts/` | 构建、检查、部署辅助脚本 | 替代正式配置 |

## 4. TypeScript 后端结构

```text
apps/api/
├── src/
│   ├── config/
│   ├── routes/
│   ├── services/
│   ├── adapters/
│   ├── health/
│   └── server.ts
├── tests/
└── package.json
```

## 5. Java Spring Boot 后端结构

```text
apps/api/
├── src/main/java/
├── src/main/resources/
├── src/test/java/
├── pom.xml
└── README.md
```

Java 项目仍必须遵守去 Mock、配置校验、接口契约和 deployment_control 评估。

## 6. simulator 规则

`simulator/` 只有在用户明确允许时才创建。创建后必须在 README 或调试记录中写明：

- 目的。
- 启动方式。
- 关闭方式。
- 删除方式。
- 是否参与生产构建。
