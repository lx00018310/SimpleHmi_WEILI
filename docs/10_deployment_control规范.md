# deployment_control 规范

deployment_control 是 SimpleHmi_WEILI 的标准部署控制能力，但不是无条件强套。

## 定位

deployment_control 负责：

- 发布包管理。
- 版本切换。
- 健康检查。
- 回滚。
- 多节点状态。
- 屏端或边缘代理协同。

deployment_control 不负责：

- 业务决策。
- 设备控制逻辑。
- 调试假数据。
- 替代业务日志。

## 适合接入

- 新项目从零构建。
- 多 HMI 节点。
- 多版本频繁发布。
- 需要回滚。
- 需要统一健康检查。
- 现场部署环境清晰。

## 谨慎接入

- 存量项目已复杂运行。
- 多硬件、多平台、多服务混杂。
- 排查链已经过长。
- 现场更适合整包覆盖。

## 接入前评估表

| 问题 | 是 | 否 |
| --- | --- | --- |
| 是否新项目？ | 倾向接入 | 谨慎 |
| 是否多节点？ | 倾向接入 | 可简化 |
| 是否需要回滚？ | 倾向接入 | 可简化 |
| 是否硬件链路复杂？ | 谨慎 | 倾向接入 |
| 是否会增加排查链？ | 谨慎 | 倾向接入 |

## 设计红线

- deployment_control 必须与业务代码解耦。
- 发布包必须可校验。
- 回滚失败风险必须写清。
- 控制器自身不应频繁升级。
- 业务模块应以基础版 + 升级包方式更新。

## 验收要求

如果接入：

- 健康检查可运行。
- 发布包可校验。
- 版本切换可追踪。
- 回滚路径明确。
- 生产包不包含未关闭 Mock UI。

如果不接入：

- 必须写明理由。
- 必须提供替代部署方式。
- 必须提供回滚方式。
