# 阶段流程规范

SimpleHmi_WEILI 的核心是阶段管控。AI 必须按阶段推进，不能把规划、架构、Demo、调试和验收混在一起。

## 总阶段

1. 规划阶段
2. 架构阶段
3. Demo 阶段
4. 开发阶段
5. 调试阶段
6. 部署阶段
7. 验收阶段

## 阶段推进规则

每个阶段必须满足四件事：

- 输入明确。
- 输出可落地。
- 禁止项已检查。
- 验收门禁通过。

门禁未通过，不得进入下一阶段；除非用户明确要求跳过，并记录风险。

## 阶段切换条件

| 当前阶段 | 进入下一阶段的条件 |
| --- | --- |
| 规划 -> 架构 | 业务目标、现场边界、硬件、外部系统、权责和失败场景已明确 |
| 架构 -> Demo | 技术栈、模块边界、接口契约、deployment_control 评估已完成 |
| Demo -> 开发 | Demo 能运行，主链路能说明，禁止项无冲突 |
| 开发 -> 调试 | 正式链路实现完成，接口契约稳定，构建通过 |
| 调试 -> 部署 | 真实联调问题已记录，临时调试能力隔离清楚 |
| 部署 -> 验收 | 发布、启动、健康检查、回滚路径已验证或标明未验证 |
| 验收 -> 归档 | 验收清单完成，未完成项明确 |

## AI 输出格式要求

AI 在每个阶段输出时必须包含：

- 当前阶段
- 已读取规范
- 已知输入
- 缺失输入
- 本阶段输出
- 不能做的内容
- 验收门禁

## 禁止行为

- 跳过规划直接设计复杂架构。
- 未判定技术栈直接写代码。
- Demo 阶段做生产级复杂功能。
- 调试阶段污染正式链路。
- 部署阶段不评估 deployment_control。
- 验收阶段只写“已完成”但不给证据。
