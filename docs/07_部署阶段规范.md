# 部署阶段规范

部署阶段负责把系统放到目标环境，并评估 deployment_control 是否接入。

## 部署前检查

- 目标系统：Windows / Linux。
- 运行方式：服务、脚本、Docker、原生进程。
- 端口规划。
- 配置路径。
- 日志路径。
- 数据目录。
- 回滚方式。
- 离线部署要求。

## deployment_control 评估

必须回答：

- 是否多节点？
- 是否需要统一版本？
- 是否需要发布包？
- 是否需要回滚？
- 是否需要健康检查？
- 是否需要屏端代理？
- 是否会增加现场排查链？

## Windows 规则

- Windows 工控机优先原生服务化部署。
- 不默认使用 WSL + Docker。
- 必须明确服务名、端口、配置路径、日志路径。

## Linux 规则

- Linux 物理机可评估 Docker。
- Docker 配置必须避免误建配置目录。
- 数据和配置必须挂载清楚。

## 禁止项

- 禁止没有回滚说明就上线。
- 禁止没有健康检查就说部署完成。
- 禁止配置文件散落多个来源。
- 禁止 deployment_control 和业务强耦合。

## 门禁

进入验收阶段前必须确认：

- 服务可启动。
- 健康检查可执行。
- 配置路径明确。
- 回滚路径明确。
- deployment_control 接入或不接入的理由明确。
