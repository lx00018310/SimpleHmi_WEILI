# 去 Mock 规范

去 Mock 是 SimpleHmi_WEILI 的核心要求。

## 默认规则

默认禁止 Mock。AI 不能主动把 Mock 当成开发主线。

正式代码必须对标：

- 真实接口。
- 真实配置。
- 真实状态。
- 真实硬件信号。
- 真实错误路径。

## 允许 Mock 的唯一条件

用户明确要求，例如：

- “做一个临时 Mock 调试页面。”
- “现在没有设备，先做独立模拟器。”
- “我允许你做 mock 接口用于演示。”

如果用户没有明确允许，AI 不得自行添加。

## Mock 隔离要求

Mock 必须同时满足：

- 独立目录，例如 `mock_temp/` 或 `simulator/`。
- 独立接口，例如 `/debug/mock/**`。
- 独立页面，不进入正式前端导航。
- 独立配置开关，生产默认关闭。
- 独立说明文档。
- 可一键删除，不影响正式系统。

## 禁止形态

- 在正式服务函数中写假数据。
- 接口失败时返回假成功。
- 前端状态异常时使用静态兜底数据伪装正常。
- 生产构建里打入 Mock 页面。
- 调试代码与业务代码混放。

## 验收检查句式

AI 验收时必须写：

```text
正式链路 Mock 检查：已检查/未检查。
检查范围：前端路由、后端路由、配置、构建产物、部署包。
结论：无 Mock / 存在隔离 Mock / 存在污染风险。
```
