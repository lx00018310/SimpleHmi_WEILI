# Demo 阶段规范

Demo 阶段只验证最小主链路。它不是最终系统，也不是功能堆叠阶段。

## Demo 目标

一个合格 Demo 必须能说明：

- 页面能打开。
- 后端能启动。
- 前后端能通信。
- 配置错误能被发现。
- 主状态能从接口进入页面。
- 没有正式链路 Mock。

## Demo 允许做

- 极简页面。
- 极简接口。
- 静态配置示例。
- 健康检查。
- 接口契约草案。
- deployment_control 的接入评估或最小占位说明。

## Demo 不做

- 不做完整权限系统。
- 不做完整报表。
- 不做复杂动画。
- 不接危险控制动作。
- 不把 Mock 当主链路。
- 不做多套技术栈兼容。

## 如果必须模拟

只有用户明确要求时，才允许建立临时模拟能力。模拟能力必须：

- 命名为临时能力。
- 放入独立目录。
- 独立路由。
- 独立开关。
- 验收时能删除。

## 门禁

进入开发阶段前必须确认：

- Demo 能运行。
- 主链路能描述清楚。
- 接口契约已初步确定。
- 无正式 Mock。
- 未完成项已列出。
