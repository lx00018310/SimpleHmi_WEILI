# deployment_control 标准能力入口

本目录当前只定义规范入口，不放任何具体业务项目的实现。

deployment_control 是 SimpleHmi_WEILI 的标准部署控制能力。新项目必须评估是否接入，但不得无脑强套。

## 标准职责

- 发布包管理。
- 版本校验。
- 服务启动和停止。
- 健康检查。
- 回滚。
- 多节点状态。
- 屏端代理或边缘代理协同。

## 非职责

- 不写业务逻辑。
- 不做设备控制决策。
- 不提供正式链路 Mock。
- 不替代业务日志。

## 推荐子目录

未来如果落代码，建议结构：

```text
deployment_control/
├── controller/
├── agent/
├── scripts/
├── native/
├── docker/
└── release/
```

## 接入判定

必须先回答：

- 是否新项目？
- 是否多节点？
- 是否需要回滚？
- 是否需要统一发布？
- 是否现场环境稳定？
- 是否会增加排查链？

## 从经验得出的红线

- 新项目可以把 deployment_control 作为部署底座逐步沉淀。
- 复杂存量项目不能因为“统一”而强行接入。
- 控制器必须和业务服务解耦。
- 发布包必须能检查是否混入未关闭 Mock。
- 回滚条件和失败风险必须写清。
