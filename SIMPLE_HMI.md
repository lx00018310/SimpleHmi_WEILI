# SimpleHmi_WEILI 工业工控前后端统一栈总规范

本文档是 AI 生成工业 HMI / 工控前后端系统时的最高优先级规范。AI 必须先读取本文档，再生成目录、代码、接口、部署与验收材料。

## 1. 项目定位

SimpleHmi_WEILI 是一个规范型开源项目，不是重代码框架。它的核心价值是把工业工控项目的标准、流程、索引、模板和禁区写清楚，让 AI 能按统一规则快速生成极简、可运行、可验收的前后端工控系统。

本仓库只沉淀通用规范和最小模板，不搬运任何具体业务项目的业务代码。

## 2. 适用范围

适用于：

- 工业 HMI
- 触摸屏一体机
- 工位操作台
- Kiosk 自助终端
- 公共工控机 + 多屏终端
- PLC、读卡器、人脸机、机器人、调度系统等外部设备或系统的前后端集成

不直接适用于：

- 纯后台管理系统
- 不涉及现场设备和部署的普通 Web 项目
- 已经高度耦合、现场运行稳定且改造风险大于收益的存量项目

## 3. AI 开发总流程

AI 必须按阶段推进：

1. 规划阶段：确认业务目标、现场边界、硬件、系统、权责、失败场景。
2. 架构阶段：判定技术栈、模块边界、接口契约、部署方式。
3. Demo 阶段：生成最小可运行演示，只验证主链路。
4. 开发阶段：按接口契约实现正式链路。
5. 调试阶段：真实联调，临时调试能力必须隔离。
6. 部署阶段：评估并接入 deployment_control 或给出不接入理由。
7. 验收阶段：按清单验证构建、运行、配置、接口、日志、回滚、无 Mock。

每个阶段必须有输入、输出、禁止项和验收门禁。

## 4. 技术栈默认规则

前端默认：

- React
- TypeScript
- Vite
- 工业极简 UI
- 生产配置外置

后端默认：

- TypeScript
- Fastify 或等价轻量 Node 服务
- 接口契约优先
- 配置启动校验

后端切换到 Java Spring Boot 的条件：

- 甲方明确要求 Java。
- 对接大型企业系统，已有 Java 生态和运维团队。
- 项目业务复杂、生命周期长、需要更强企业级治理。
- 现有系统以 Spring Boot / Java 为主，复用成本明显低于迁移成本。

AI 不得静默选择后端技术栈。必须在架构阶段给出判定理由。

## 5. 去 Mock 最高红线

默认禁止 Mock。

正式业务链路必须对标真实接口、真实状态、真实硬件信号。只有用户明确要求“做一个 Mock 调试能力”时，AI 才能添加临时 Mock。

临时 Mock 必须满足：

- 独立目录。
- 独立接口。
- 独立入口。
- 独立开关。
- 不进入正式路由。
- 不参与生产打包。
- 可一键删除。

验收阶段必须检查正式链路中没有 Mock、假成功、静默降级和调试入口残留。

## 6. deployment_control 定位

deployment_control 是 SimpleHmi_WEILI 的标准部署控制能力，必须有规范和模板入口。

新项目必须评估是否接入 deployment_control。默认建议纳入评估；是否启用由项目复杂度、现场环境、维护方式和风险决定。

适合接入：

- 新项目从零构建。
- 公共工控机或多 HMI 需要统一发布。
- 需要版本包、健康检查、回滚、自愈。
- 现场部署环境相对稳定。

必须谨慎：

- 复杂存量项目。
- 多硬件、多系统、多运行环境混杂。
- 现场排查链已经很长。
- 直接整包覆盖更可靠。

AI 不得把 deployment_control 当成万能方案。

## 7. 配置原则

- 一个配置源解决一个问题。
- 生产配置必须外置。
- 占位值、空令牌、错误 IP、非法端口必须拒绝启动。
- 前端只保留必要运行时配置。
- 后端负责强校验和清晰错误提示。

## 8. 调试隔离原则

调试能力必须服务于正式场景还原，而不是制造脱离现场语义的假数据。

允许：

- 独立调试接口。
- 独立调试页面。
- 独立 simulator。
- 有明确开关和权限。

禁止：

- 正式接口里混入调试逻辑。
- 正式页面里保留调试按钮。
- 生产环境默认打开调试能力。
- 联调完成后无法删除调试代码。

## 9. 验收原则

任何“完成”都必须有证据：

- 构建通过。
- 服务能启动。
- 错误配置能拒启。
- 接口契约能验证。
- 正式链路无 Mock。
- deployment_control 评估完成。
- 文档与实际实现一致。

未验证的内容必须写为“未验证”，不得写成“已完成”。

## 10. 工业项目通用教训

- 从零新项目可以逐步沉淀 deployment_control、发布包、健康检查、回滚、自愈。
- 复杂存量项目强套重型部署控制会增加排查链，应优先保持生产链路薄。
- Mock、调试、配置和部署如果不隔离，后期会成为主要风险源。
