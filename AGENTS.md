# SimpleHmi_WEILI AI 工作入口

本仓库是规范型开源项目。AI 的任务不是先写大量代码，而是按本文档索引读取规范、选择阶段、生成极简可用的工业 HMI / 工控前后端系统。

## 必读顺序

任何任务开始前，AI 必须按顺序读取：

1. `SIMPLE_HMI.md`：总规范，最高优先级。
2. `docs/00_INDEX.md`：规范索引和阶段入口。
3. `ENGINES.md`：前后端技术栈判定规则。
4. 当前阶段对应规范文件。
5. 当前阶段对应模板文件。

如果用户没有明确当前阶段，AI 必须先判断阶段；判断不清时先提问，不得直接进入实现。

## 总原则

- 先读规范，再写代码。
- 先收口范围，再做实现。
- 默认生产对标，默认禁止 Mock。
- deployment_control 是标准能力，但新项目必须先评估是否需要接入。
- 复杂存量项目、多硬件混合项目、现场排查链过长的项目，不得无脑强套重型部署控制。
- 每一步输出必须可验证；无法验证时必须明确写出未验证项。

## 禁止项

- 禁止在正式业务链路内写 Mock、假成功、静默降级。
- 禁止不做技术栈判定就默认选型。
- 禁止把任何具体业务项目的代码直接搬入本仓库。
- 禁止把调试页面、调试接口混入正式前端路由或正式 API。
- 禁止生成没有验收清单的“完成”结论。
